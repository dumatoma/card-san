## 1. プロダクト概要

### 1.1 プロダクトビジョン

Card-San（カードサン）は、日本のローカル生活サービス店舗（美容院・ネイルサロン・フィットネス・飲食店等）向けのオールインワン デジタル会員管理プラットフォームである。

- **消費者（C端）**: スマートフォンで複数店舗の電子会員カードを一元管理し、クーポン・予約・チャットを利用できる
- **店舗オーナー/スタッフ（B端）**: 顧客管理（CRM）・マーケティング通知・メニュー・予約管理・会員カードデザイン・サブスクリプション・決済を一括管理できる

### 1.2 ターゲットユーザー

| ユーザー種別 | 説明 |
|-------------|------|
| 消費者 | 日本在住、複数店舗の会員カードをスマホで管理したいユーザー |
| 店舗オーナー | 美容・ネイル・フィットネス・飲食等のローカル店舗経営者 |
| 店舗スタッフ | 顧客対応・予約管理を担当するスタッフ |
| システム管理者 | Card-Sanプラットフォーム全体の管理者（admin使用） |

### 1.3 リポジトリ構成

| リポジトリ | 内容 |
|-----------|------|
| `card-san` | フロントエンド（uni-app 多端 + admin 管理画面） |
| `api.card-san.jp` | バックエンドAPI（Laravel 8、PHP） |

### 3.3 jpRegister — 新規店舗登録

**AppID**: `__UNI__ED993F2`  
**対象**: H5（ブラウザ）+ 各種スマホプラットフォーム

#### 機能・フロー

3ステップ登録フロー:

1. **Step 1 / 3** — メールアドレス入力・送信
2. **Step 2 / 3** — メール認証コード（6桁）入力・再送信対応
3. **Step 3 / 3** — 氏名・店舗名入力・利用規約同意
4. **完了画面** — アカウントID表示・CardSan管理アプリダウンロード案内

**追加ページ**: 利用規約・プライバシーポリシー表示

### 3.4 outsiteAppointment — 外部リンク簡易予約

**AppID**: `__UNI__26F5B98`  
**対象**: H5（店舗ページの外部リンクから遷移）

#### 機能・フロー

5ページで完結する軽量予約フロー:

1. **メニュー一覧** — サービスをカテゴリ別に表示・選択・お気に入り

4. **予約内容確認** — 来店時間・メニュー・担当者・指名料
5. **予約完了** — 予約番号・詳細表示

**コンポーネント**: カスタム日本語カレンダー・週次ビュー・ドラッグ&ドロップ並び替え

## 4. バックエンドAPI仕様概要

**本番ドメイン**: `https://api.card-san.jp`  
**フレームワーク**: Laravel 8（PHP）

### APIパス設計

| パスプレフィックス | 対象 |
|-------------------|------|
| `/api/member/*` | 会員端（jpcard・outsiteAppointment） |
| `/api/shop/*` | 店舗管理端（jpshop・jpRegister） |
| `/adminapi/*` | システム管理（admin） |
| `/api/config` | 共通設定取得 |

### 認証方式

- `token` + `uuid` をリクエストヘッダーに付与
- Content-Type: `application/json;charset=UTF-8`

### リアルタイム通信

- WebSocket接続先: `ws://47.245.1.2:21000`
- 接続後に `client_id` をバインド（`/api/member/bind` or `/api/shop/bind`）
